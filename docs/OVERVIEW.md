# RIO System — One Page Overview

## Core Idea

A system that observes and reflects before action, while preserving full human control.

---

## System Flow

text Task  ↓ Pattern Selection  ↓ Draft / Proposal  ↓ MANTIS (Contrast)  ↓ Trigger Engine  ↓ Surface (optional)  ↓ Logging  ↓ Trend Analysis  ↓ Human Decision  ↓ RIO Governance (external)  ↓ Execution + Receipt 

---

## What Each Layer Does

| Layer | Purpose |
|------|--------|
| Pattern Corpus | Stores known behavioral patterns (reference) |
| Selector | Chooses relevant patterns for a task |
| MANTIS | Compares current state to patterns |
| Trigger | Decides when to surface contrast |
| Surface | Shows findings to human (non-blocking) |
| Logging | Records what happened |
| Trends | Detects patterns over time |
| RIO Protocol | Enforces execution + proof (external) |

---

## Key Properties

- Non-authoritative (never executes or decides)
- Deterministic (no guessing)
- Append-only memory (no rewriting history)
- Fail-closed execution (handled by RIO)

---

## Mental Model

text Patterns → What tends to happen MANTIS → What is happening Trends → What keeps happening Human → What should happen 

---

## One Line

Observe → Surface → Decide → Prove
