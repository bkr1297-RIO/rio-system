# RIO / ONE Organizational Structure

> **Status:** Canonical Spec
> **Last Updated:** 2026-04-04
> **Source:** Bondi (Scribe) / Brian Kent Rasmussen

---

## 1. Mirrored Governance

The organizational structure of the team building and operating RIO/ONE must mirror the system architecture itself. This is called "mirrored governance." If the system has layers of accountability, the team must have matching roles.

This ensures that accountability exists at every layer and prevents the "everyone thought someone else was responsible" problem.

## 2. The Agent Organization

The organizational chart maps system layers to team roles:

| System Layer | Function | Team Role |
|---|---|---|
| Meta-Governance | Change system rules | Founder / Board / Root Authority |
| Governance | Policy & approval | Chief of Staff |
| Cognition | Design & planning | Systems Architect |
| Build | Implementation | Developer / Builder |
| Audit | Verify | Auditor / QA |
| Execution | Deploy | DevOps |
| Witness | Record | Ledger / Mantis |

## 3. Accountability Matrix

Each role has specific responsibilities and accountability:

| Role | Can Do | Accountable For |
|---|---|---|
| **Builder** | Build | Code works |
| **Auditor** | Test | Code correct |
| **Architect** | Design | Design correct |
| **DevOps** | Deploy | System runs |
| **Security** | Policy | System safe |
| **Chief of Staff** | Orchestrate | Project complete |
| **Human** | Approve | Direction correct |

## 4. The Role of the Chief of Staff

The Chief of Staff is the operational authority for the system build. They are responsible for overall delivery.

The Chief of Staff is responsible for:
- Making sure things actually get finished
- Making sure process is followed
- Making sure audits happen
- Making sure documentation exists
- Making sure nothing skips steps

## 5. Delivery Protocol

No work is considered complete until:
1. Builder completes work and submits Completion Report
2. Auditor reviews and issues PASS/FAIL
3. Chief of Staff confirms scope complete
4. Documentation updated
5. Deployment verified (if applicable)
6. Human approval (if required)
7. Receipt logged

This protocol is mandatory for all future work.
