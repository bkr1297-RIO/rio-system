# State-Aware Governance (v1)

## Overview

This extension introduces structured state into governance evaluation.

## Flow

State → Integrity → Risk → Policy → Approval → Execution → Receipt

## Rules

- State must be explicit
- State is non-authoritative
- State must not trigger actions
- RIO evaluates only provided inputs

## Example

High activation + medium risk → require approval

## Guarantee

State informs decisions but never determines them.
